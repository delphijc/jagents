# WORKFLOW: Quick Flow

**Type:** BMM Workflow
**Agents involved:** Quick Flow Solo Dev (Barry)

---

## Purpose

A fast-track development workflow for bugs, small features, or prototypes where the full BMM overhead is unnecessary.

## Steps

1. **Create Tech Spec (`*create-tech-spec`):** Barry analyzes the request and creates a lightweight technical specification.
2. **Quick Dev (`*quick-dev`):** Barry implements the specification end-to-end.
3. **Code Review (`*code-review`):** Barry performs a self-review or requests a review.

## Trigger

- Command: `*quick-flow`
- Context: Small scope items.

## Outputs

- Tech Spec
- Implemented Code
